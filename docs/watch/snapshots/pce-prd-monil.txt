Executive Summary  Post   Course evaluation   (PCE)   is a non - discretionary accreditation requirement for every health  profession program Exxat   serves. The demand already exists within the installed base   —   yet clients  are currently fulfilling it through third - party platforms, leaving a meaningful and recoverable gap in  per - account revenue. This document does not make the case for whether to build ; that decision has  been made. It addresses how to build with intention   —   defining what is required to meet market  expectations, what will differentiate Prism from established incumbents, and what will make it the  platform clients choose to stay on.  This is a master PRD that covers following requirements:  1.   Create Template  2.   Push Survey end to end  2. Problem Statement  Health professions programs are obligated to run Course evaluations every semester, yet the tools  currently available to them create more administrative burden than they resolve. Four distinct  problems characterize the current state.  Problem   Current State & Impact  Data that exists but  cannot be acted upon  Faculty   Engagement with results is very low . In customer discovery,  one faculty noted they had never used their platform's reporting  feature because a single report required 10+ steps to generate .
Disconnected data  streams  Course evaluation data and clinical competency data live in  separate systems . No platform can currently connect how cohort  groups rated a course to how they performed in the corresponding  clinical rotation.  No single system to log  actions  For Ex: PT has CAPTE Standard 2C that requires a documented  cha in: result → action → reassessment. Today this is assembled  manually across emails and spreadsheets at the point of   self - study,  under time pressure.  Longitudinal   Analytics  Explorance   B lue just shares Survey reports to each faculty   at the  end of course   as one   isolated   report. To study longitudinal data,  Faculty would have to manually run some excel formulaes or use AI  tools for that  Correlation   between Didactic Course evaluation and Clinical course class performance:  Key   Product D ecisions   for Phase 1:  1. No IP  address, device ID, or response timing metadata exposed. FERPA  compliant . Suppression rule visible to students as a  trust signal before submission  Scoring standardisation   1 – 5 Liker t scale enforced as standard across all evaluations. Anchor labels  standardised .
