Overview  This document covers the Assessment Builder and Assessment Distribution modules for the  exam management platform. The Assessment Builder enables faculty to construct robust exams  with sections, timers, and specific mark configurations by pulling items from   the Question Bank.  The Distribution component governs how those exams are scheduled, configured with specific  delivery settings (including student accommodations), and pushed to students.  2. Goals  Provide a modern, intuitive, and "full - bleed" SaaS interface for exam creation with AI assistance.
Support complex exam structures including sections, dynamic timers, and question reordering.  Facilitate secure and flexible assessment delivery, including granular student accommodations.  Enable seamless cross - faculty collaboration with clear "Submit for Review" workflows.  3. Assessment Builder (Creation & Structure)  3.1 Lifecycle Stepper  To guide faculty through the assessment lifecycle, the builder will employ an ultra - slim  horizontal stepper with the following 6 states:  Draft: Initial creation, metadata definition.  Building: Adding sections, pulling questions from the Question Bank, setting marks.  Review: Submitted for peer or department head review.  Published: Approved and finalized.  Scheduled: Assigned to cohorts with delivery dates.  Live: Exam is currently accessible to students.  3.2 Metadata & Settings  The overview interface will utilize a "Meta Strip" (horizontal key - value chips) summarizing:  Assessment Name  Type (Tag): e.g., Midterm, Final, Quiz  Weightage (Tag)  Delivery: e.g., On - site, Remote, Lockdown Browser  Duration: Total time (auto - calculated from sections or manually overridden)  Total Marks: Auto - calculated badge  Scheduled Date  3.3 Structure & Mark Distribution  Tabs Navigation: The builder will be organized into 5 primary tabs: Structure, All Questions,  Collaboration, Settings, Pre - Read.  Sections: Faculty can create sections (e.g., Section A: MCQ, Section B: Essays). Each section  can have its own timer, question ordering rules (randomized vs fixed), and specific instructions.
Mark Distribution Card: A visual statistics grid displaying the weightage of marks across topics,  Bloom's Taxonomy levels, and difficulty to ensure a balanced exam.  3.4 All Questions Management  Interface to add questions via the Question Bank   (with and without AI assistance).  Drag - and - drop reordering.  Warning indicators for questions that lack required tags or have unresolved Draft statuses.  4. Collaboration Workflow  Shared Ownership: Faculty can invite other users (Course Directors, Contributors) to co - author  an assessment.  Submit for Review: An explicit action to send the built assessment to a Department Head or  Outcome Director for approval.  Versioning Interaction: Assessments pin to the specific version of a question used at the time of  creation to prevent unexpected changes if the master question is modified later.  5. Assessment Distribution & Settings  5.1 Pre - Read & Instructions  Assessment - level Instructions: Configurable text/media shown before the timer begins.  Section - level Pre - read: Configurable context provided before entering a specific section, which  pauses the main assessment timer.  5.2 Student Accommodations  Administrators and Course Directors can define specific overrides for students with special  requirements. This features a Student Accommodations Modal allowing:  Extra Time: Add +N minutes or a fixed multiplier (e.g., 1.5x time).  Scribe   Required: For injured / differently abled students who need physical assistance.  Extended Breaks: Configuration for paused intervals.  Screen Reader Mode: Enforced or optional accessibility features.  Bulk Apply: Ability to select multiple students and apply uniform accommodations  simultaneously.  5.3 Delivery & Scheduling  Assign the assessment to specific Course Cohorts or individual student lists.
Define Open and Close windows.  Configure submission rules (e.g., auto - submit on timer expiry vs hard block on unanswered  questions).  Setup backward navigation blocks (preventing students from returning to previous questions).  6. Roles & Permissions  System Admin / Dept Head: Full override rights, approval authority.  Course Director: Can publish and schedule assessments for their courses.  Contributor / Faculty: Can build and edit draft assessments but may require review before  scheduling.
