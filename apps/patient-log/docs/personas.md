# Patient Log — Personas

⚠️ **Empty placeholder.** No personas have been confirmed for Patient Log yet. The intake skill will populate this file when meeting context, user research, or decisions identify real personas.

## Template (do not treat as real)

When the first persona is confirmed, write it using this shape:

```markdown
## P<N> — <Persona name>

**Who:** <one-sentence role>
**Phase 1 surfaces:** <list of screens / flows>

**Goals**
- <goal 1>
- <goal 2>

**Frustrations (current state)**
- <frustration 1>

**Jobs to be done**
- <JTBD 1>
- <JTBD 2>
```

## How to add a persona

1. Reference a meeting or user research in conversation — the intake skill pulls from Granola.
2. Confirm the persona through the intake flow.
3. The skill writes here, with provenance frontmatter (which meeting / decision / source).

## What NOT to do

- Don't invent personas based on product name guesses.
- Don't copy-paste personas from other Exxat products without validation — Patient Log's user model may differ from PCE / Exam Management.
- Don't write aspirational personas ("we hope nurses use this"). Write real personas confirmed by user research, customer interviews, or stakeholder decisions.
