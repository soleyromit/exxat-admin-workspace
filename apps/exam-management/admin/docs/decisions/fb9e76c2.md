## User Flows

- **Faculty creates assessment:** Starts from topics/content areas (not course objectives directly). Selects which content topics to cover, decides distribution (10 questions on X, 5 on Y), picks from QB. Faculty assumption: "I taught it, therefore I have questions for it."
  > "One big clarity I got from last 10 minutes of a conversation is that faculty may not create assessments with course objectives as starting point. It will most likely be course content as a starting point. Topics as a starting point."

- **Curricular assessment loop (4 phases):** Must teach (standards → curriculum) → Am teaching (course objectives → courses) → Am assessing (questions → assessments) → Students learning (scores → feedback to curriculum)
  > "You design a curriculum because it meets certain standards. Content delivery — you teach your students via course objectives. Then you design questions to test student understanding. Then based on the scores, you get insights into whether your student learned it."

---

## Design Decisions

- **AI gap analysis: assessment level >> question bank level:**
  > "Finding out gaps in assessment is more important than finding out gaps in your question bank because you may have a very well covered question bank, but you are not picking up those questions and using them in an assessment, it is as good as you've not created those questions."

- **Tags/labels = single consistent concept (like Gmail labels):** Do NOT use "categories" (competitor term). Do NOT create two separate capabilities (attributes vs. direct mapping) — that created confusion in curriculum mapping. One system, one way to create links.
  > "Something like labels which you see in Gmail. You could have nested labels. [...] I want to try and avoid that in exam management. There is only one way to create links."

- **Question mapping Phase 1 = direct to standards:** Allow questions to be directly mapped to standards/competencies in the QB, independent of whether the user has curriculum mapping set up.
  > "We definitely want to give the ability for a user to map the questions that they are creating or importing in our exam management module directly to the standards."

- **Question mapping Phase 2 = via course objectives:** Map question → course objective → standard (completing the curricular loop). Only possible if they use Prism curriculum mapping.
  > "If you want to really create a differentiated layer that we are able to give you that additional layer of completing your curricular assessment loop, you should be able to map because your course objectives are already mapped to standards."

- **Gap analysis for published standards (no curriculum needed):** For programs with published blueprints (NAPLEX, NCLEX, NCCPA), EM can show gaps between their questions and the standard blueprint — no curriculum mapping required.
  > "For all published content area standards like the NAPLEX blueprint or the NCLEX for nursing, we already know what students need to appear for their board exam. If across all your courses, your entire curriculum, we see assessment gap, that we can easily highlight."

- **Faculty very reluctant to add new unvalidated questions:** 99.9% already have existing validated question banks. PBI (point biserial index) and discrimination coefficient are key quality signals. Faculty recycle 5–15% of questions per exam.
  > "99.9% of our users already have their question banks. Already have their validated assessments. They are not going to create new questions in lump sum because they don't know how these new questions are going to perform."

- **AI should not replace human tagging — it assists:** AI can SUGGEST mappings to standards based on question stem content, but human can override. AI-generated questions from PowerPoint/LMS content also a valid use case.

---

## Scope Constraints

- **IN (Phase 1):** Question → standard direct mapping; assessment-level gap analysis vs. standards blueprints
- **IN (Phase 2+):** Question → course objective → standard chain (requires curriculum mapping integration)
- **IN (Phase 2+):** Question bank gap analysis (lower priority than assessment gap)
  > "If we can only do one thing, then we introduce it in the assessment gap rather than question bank gap would be my recommendation."
- **DEFERRED:** AI-proof question types (human-judgment-testing questions) — 2027–2028 functionality
  > "I look at it as more a 2027–2028 functionality. This is so evolving we don't have enough context to build a solution."

---

## Data / Entity Rules

- **Label taxonomy:** Program-level labels (admin-defined, all faculty use these) + user-level personal labels (per faculty)
  > "A program can define certain categories that they expect all users, all faculty of all courses to map their questions to. Each user also gets capability to create certain tags that are only visible for them."
- **Standard ≠ label:** Standards are a type of label/mapping target. Don't call the whole system "standards" — too narrow.
